NP-Completeness Proof

NP-COMPLETE

Step 1: Dominating Set ∈ NP

Certificate: A subset D ⊆ V with |D| ≤ k.

Verifier (polynomial time O(|V| + |E|)):

  • Check |D| ≤ k. If not, reject. (O(k))
  • For each vertex v ∈ V, check whether v ∈ D or ∃ u ∈ D such that (u,v) ∈ E. (O(|V|·|E|) → O(|V|+|E|) with adjacency list)
  • If every vertex is covered, accept. Otherwise, reject.

Since verification runs in polynomial time, Dominating Set ∈ NP. ∎

Step 2: Vertex Cover ≤ₚ Dominating Set

We show Vertex Cover (known NP-complete) reduces to Dominating Set in polynomial time.

Construction: Given a Vertex Cover instance (G = (V, E), k), construct graph G' = (V', E'):

  • Start with V' = V, E' = E.
  • For each edge (u, v) ∈ E, add a new gadget vertex w_{uv} connected only to u and v.

The parameter remains k. This gives (G', k).

Polynomiality: We add at most |E| new vertices and 2|E| edges → |V'| = |V| + |E|, which is polynomial.

Reduction Diagram

Each original edge (u,v) gets a private gadget vertex w_{uv}. The gadget vertex can only be dominated by u or v — forcing at least one of them into any dominating set. This mirrors the vertex cover constraint.

Forward Direction: VC → DS

Claim: If G has a vertex cover C with |C| ≤ k, then G' has a dominating set D with |D| ≤ k.

Proof: Set D = C. We verify every vertex in V' is dominated:

  • Original vertices in C: trivially in D, so dominated. ✓
  • Original vertices not in C: since C is a vertex cover, every edge (u,v) has u ∈ C or v ∈ C. If v ∉ C, there must be some neighbor u ∈ C = D (as an isolated vertex would have no edge to cover anyway). ✓
  • Gadget vertex w_{uv}: since C covers edge (u,v), at least one of u, v ∈ C = D. Since w_{uv} is adjacent to both, it is dominated. ✓

So D = C is a dominating set in G' of size ≤ k. ∎

Backward Direction: DS → VC

Claim: If G' has a dominating set D' with |D'| ≤ k, then G has a vertex cover of size ≤ k.

Proof: We may assume D' ⊆ V (original vertices only), because:

  • Any gadget vertex w_{uv} ∈ D' only dominates itself and u, v. Replacing w_{uv} with either u or v still dominates w_{uv} (now as a neighbor) and at least as many others. So we can always swap gadget vertices out for original vertices without increasing |D'|.

Now let C = D' ∩ V. Claim: C is a vertex cover of G.

  • Take any edge (u,v) ∈ E. The gadget vertex w_{uv} must be dominated in G'. Its only neighbors are u and v. So either w_{uv} ∈ D' (impossible after our swap), or u ∈ D', or v ∈ D'.
  • Therefore u ∈ C or v ∈ C for every edge — C is a vertex cover. ✓

Since |C| ≤ |D'| ≤ k, we have a vertex cover of size ≤ k. ∎

Conclusion

  • Dominating Set ∈ NP (Step 1).
  • Vertex Cover ≤ₚ Dominating Set (Step 2): polynomial reduction, both directions proved.
  • Vertex Cover is NP-complete (Garey & Johnson 1979).
  • Therefore Dominating Set is NP-complete. ∎

Problem Overview

What is the Dominating Set Problem? ▼

Given an undirected graph G = (V, E) and integer k, does there exist a subset D ⊆ V with |D| ≤ k such that every vertex in V is either in D or adjacent to a vertex in D?

Dominating Set is NP-COMPLETE — proven via reduction from Vertex Cover.

How does it differ from Vertex Cover? ▼
PropertyVertex CoverDominating Set
GoalTouch every edgeBe "close" to every vertex
Constraint∀(u,v)∈E: u∈C or v∈C∀v∈V: v∈D or ∃u∈N(v)∩D
Isolated verticesAlways coveredMust be in D
Real-world Applications ▼
  • Wireless networks: Place base stations (dominating set) so every device is within one hop.
  • Social influence: Select minimal set of influencers so every person knows at least one.
  • Sensor networks: Cover every node with at least one sensor nearby.
  • Facility location: Place facilities so every client is near one.
Worked Example: Greedy Algorithm ▼

Consider a star graph K₁,₄ with center node 0 and leaves 1, 2, 3, 4.

Step 1: Compute domination gain for each vertex:

  • Node 0: dominates itself + 4 leaves = 5 vertices
  • Nodes 1–4: each dominates itself + node 0 = 2 vertices

Step 2: Select node 0 (max gain = 5). All 5 vertices are now dominated.

Result: D = {0}, |D| = 1. This is optimal. ✓

The greedy always picks the "most useful" vertex — the one dominating the most currently-undominated vertices.

Greedy Approximation Ratio (CLO-3) ▼

The greedy domination algorithm is a special case of Set Cover (each vertex v covers the set {v} ∪ N(v)). The greedy set cover achieves approximation ratio H(Δ+1) = O(ln Δ), where Δ is the max degree.

In general: greedy gives a O(ln n) approximation of the optimal dominating set size.

Counterexample (greedy not optimal): Consider a path 1–2–3–4–5–6. Optimal D = {2, 5} (size 2). Greedy may pick 1 first (if tiebreak), leading to D = {1, 4} or similar of size 2 — acceptable here — but on larger carefully crafted graphs, greedy can exceed optimal by O(ln n) factor.

🔑 Key Concept: Why NP-Complete?

No polynomial-time algorithm for Dominating Set is known. The reduction from Vertex Cover shows that if we could solve Dominating Set in polynomial time, we could solve Vertex Cover (and all NP problems) in polynomial time — implying P = NP.

🔑 Key Concept: The Gadget Trick

The edge-subdivision gadget w_uv is a "sentinel" — it forces either u or v to enter any dominating set (since w_uv has no other neighbors). This perfectly encodes the vertex cover constraint inside the dominating set framework.

🔑 Key Concept: Greedy as Set Cover

Each vertex v can be seen as a "set" covering {v} ∪ N(v). Finding a minimum dominating set is equivalent to the minimum Set Cover problem on these sets. Greedy Set Cover achieves a ln(n)+1 approximation ratio.
